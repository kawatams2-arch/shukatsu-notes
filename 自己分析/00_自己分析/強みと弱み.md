# 強みと弱み

## 💪 強み
- **自覚している強み**:論理的思考力 / 構造化思考(イシュー思考) / 学習サイクルの速さ
- **他者から見えている強み**(本人は気づきにくい):素直さ / 人間関係構築力(縦横問わず)
- 一言で:**課題を構造化し、継続できる仕組みを設計して成果を出す力**(論理 × 独創性 × 仕組み化)

裏付けエピソード: [[増量]] (自分への仕組み化) / [[Cチームリーダー]] (他者・組織への仕組み化)

## ⚠️ 弱み
**端的に言うと「頭でっかちになりやすい」**

- 物事の因果関係が明確に見えないと、その価値を過小評価してしまう
- 「AならばB」の道筋が立たないものは「やる意味がない」と行動前に結論を下し、可能性を切り捨てていた
- 納得できないと気持ちが乗らない
- コンフォートゾーンから抜け出しにくい(本来は安定思考)

### 具体例(壁当てエピソード)
入部当初、基礎技術が伸び悩む中、先輩から「理屈はいいから毎日壁に向かってボールを投げ、感覚を体に染み込ませろ」と助言をもらった。当時の私はそれを「根性論」だと決めつけ、「原理を理解せず数をこなしても変な癖がつくだけ。まず理想的なフォームを理論的に解明するのが先決」と考え、自らの論理で反復練習の不足を正当化した。半年後、壁当てを継続した同期との技術差は歴然だった。**頭で否定したものに、自分の成長機会が眠っていた。**

### 改善への取り組み(現在進行形)
「この練習は無駄かもしれない」と結論を出して終わるのではなく、「無駄かどうかを、まず少しの実践で試す」**検証作業に置き換え、思考に期限を設ける**。→ 行動習慣としての「まず試す」

## 🔄 強みと弱みは表裏一体
| 強みの面 | 弱みの面 |
|---|---|
| 因果が明確 → 継続できる | 因果が見えない → 動けない |
| ロジックで判断できる | 論理で説明できないものを排除する |
| 納得すれば切り替えが早い | 納得しないと気持ちが乗らない |
| 意識的に挑戦を選べる | コンフォートゾーンを出るのは苦手 |

面接での語り方: 弱みは「克服済み」ではなく「**意識して改善中**」と表現する。
